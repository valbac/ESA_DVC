# ESA_DVC

## 1. Repository erstellen
Erstellen Sie sich ein Repository in GitHub oder GitLab.

## 2. Eigenes Projekt hochladen
Pushen Sie ein eigenes Projekt hoch (z. B. das CCD-Projekt) oder erstellen Sie ein neues Projekt.

## 3. Git-Methoden anwenden
Wenden Sie alle in der Lerneinheit genannten relevanten Git-Methoden beweisbar an (das GitHub-Repository dient als Nachweis):

- `push`
- `pull`
- `add`
- `commit`
- `diff`
- `status`
- `rm`
- `mv`
- etc.

## 4. Mit Zeitreisen experimentieren
Experimentieren Sie mit Git-Zeitreisen (z. B. ältere Commits ansehen oder zurücksetzen).

## 5. Branches erstellen und mergen
Erstellen Sie zwei unterschiedliche, aber ähnliche Branches.

- Wechseln Sie zwischen den Branches hin und her.
- Mergen Sie die Branches anschließend wieder zusammen.

## 6. Pull-Request erstellen
Erstellen Sie in GitHub einen Pull-Request mit Bezug auf:

https://github.com/edlich/education

Der Beitrag kann klein, nützlich oder witzig sein, soll aber **nicht über die Shell**, sondern direkt über die GitHub-Oberfläche erstellt werden.
